Battle Heros

A Firstperson Shooter Game (PVP - Multiplayer), Where the player who gets the highest kill count in a match wins.

Project Description

Genre : Action, Shooter

Target Platform : Android

Role : Level Designer

My Work

As a level designer, I focused on creating multiplayer maps for Battle Heroes. Each map needed spawn points for 10 players, and the level design was built around that requirement. I also participated in playtesting to ensure everything functioned as intended.

This was the first project where I experimented with Unity’s optimization tools. The assets used to build the level were sourced from the Unity Asset Store, and several underwent optimization, such as polygon reduction.

For example, some asset packages included multiple houses with interiors, which made the models heavier to render. By removing the interiors, we reduced the polygon count and improved frame rates.

We also implemented various other optimizations to enhance performance without sacrificing too much visual quality. This project helped me better understand the importance of level metrics, scale, and optimization in game development.

Match Blast

A match-3 puzzle game where the player must defeat all the zombies to win the level.

Project Description

Genre : Puzzle

Target Platform : Android

Role : Game Designer and Programmer

My Work

This project allowed my internship employer to test my programming skills in Unity. During this project, I implemented a two-dimensional array, a concept I had first learned during my C# studies, and used it to set up the intended functionality.

Since I didn’t know how to apply a three-dimensional array in the game, I looked for an online tutorial to figure out how I could implement the concept. The code I found in a tutorial was for a 2D array, and I adapted it to work in 3D.

I modified the code based on my understanding of the concept. While the result looked like what I wanted, its behavior was different from what I expected. After a few iterations and some help from a colleague, I was able to get the code running as intended.

It was a fun endeavor, and the experience sparked my interest in prototyping games—especially during the early stages of development.

Island Escape

Escape the island with the help of a slingshot, but beware of roaming pirates.

Project Description

Genre : Action, Hyper-Casual

Target Platform : Android

Role : Game Designer, Level Designer and Programmer.

My Work

Island Escape is a game that was selected by the COO of Ether Games from a few concepts I had pitched, including a prototype. Initially, it was designed as a typical hyper-casual game where the player needed to collect parts to fix a small boat and escape the island. At that stage, the COO wasn’t particularly interested.

Then I had a new idea: the character would try to escape the island by throwing a stone at passing ships. The ships could either be standard cargo ships or pirate ships, but this wouldn’t be revealed until the ship responded to the player's distress signal (i.e., after the stone hit the ship). That twist made the concept more interesting — and it turned out that one of my colleagues had already developed a version of the game with a small variation.

One day, that colleague shared some files with me, and the COO asked me to re-create the game. He also shared a gameplay video for reference. I took this opportunity to explore some programming concepts — I broke things, fixed them, and eventually made it work.

Shadow Master

A hyper-casual game where players must rebuild a missing part of a walking path by matching shadows cast by random objects to complete the path.

Project Description

Genre : Hyper-Casual

Target Platform : Android

Role : Level Designer and Programmer.

My Work

This concept was developed internally. I first created a prototype based on an idea that came to mind, and it was initially set in a 3D world.

When I began developing the prototype, I struggled to find a good camera system. I couldn’t figure out a camera angle that would work well for both touch controls and overall gameplay visibility. After a brief discussion with my colleague, he mentioned that he had set up the game using a Side-Scroller perspective. It turned out to be the perfect choice for this type of game, so I switched the perspective to a side-scrolling (2D) view.

Weapons Inc

Do you like flashy weapons? How about creating one? Choose your weapon, mix it with an element, test it, and sell it for a price. Weapon Inc. enters the world of elemental weapons.

Project Description

Genre : Hyper-Casual, Action, Shooter

Target Platform : Android

Role : Level Designer

My Work

The game was published and began collecting data from users. I was tasked with making changes to the game based on this data. This is another project where I led a small team.

Weapons Inc. was a project that made me truly appreciate the value of having solid technical knowledge in game development — especially during critical moments.

The game was plagued by a persistent hard crash, and neither I nor my team could figure out what was going wrong. The game had integrated an SDK from a third party, and I suspected it might be causing a memory leak. The SDK provider’s website even included a warning that improper usage could lead to such issues, though we had no concrete evidence to confirm it.

Based on this assumption, I rewrote the C# code to better manage memory and prevent potential leaks. After several rounds of rewriting and testing, the mysterious crash was finally resolved — thanks to my understanding of the Unity engine and the C# language. The game is now stable and live on Google Play.

RiddleMath

Want to test your calculating skills? Look no further than RiddleMath! It’s a fun and engaging cross-math puzzle game that’s quick and easy to play anytime, anywhere.

Project Description

Genre : Puzzle

Target Platform : Android

Role : Level Designer

My Work

This is the first game in which I used a Custom Level Generator Tool to quickly create Cross-Levels. As a level designer, I applied my basic mathematical skills to develop these levels.

The tool made my work much easier and gave me more time to playtest my creations. On top of that, it included additional features that allowed us to generate cross-levels with varying difficulty.

Not only did the tool generate the cross-levels, but it also created math equations for them. I used a mix of auto-generated equations and custom ones that I created myself.

While working on RiddleMath, my Senior Designer introduced me to the concept of Primed Levels — levels that are carefully handcrafted by designers to deliver a specific experience to the player. This helped shape the overall quality and intention behind the level design.

The game is currently live on Google Play.
